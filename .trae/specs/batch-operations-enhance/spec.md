# 批量操作增强 Spec

## Why
当前批量管理仅支持删除操作，用户在批量选择多个提示词或技能后，无法进行分类修改、标签管理、置顶切换等常用批量操作，需要频繁逐个编辑，效率低下。

## What Changes
- 提示词模块 batch-bar 新增"更多操作"下拉菜单，包含：修改分类、添加标签、移除标签、置顶、取消置顶
- 技能模块 batch-bar 新增"更多操作"下拉菜单，包含：添加标签、移除标签、置顶、取消置顶
- 后端新增 7 个批量更新 API 方法
- 前端复用现有 Modal/Confirm 组件实现交互弹窗

## Impact
- Affected specs: 无破坏性变更
- Affected code: service/prompt.go, service/skill.go, handler/prompt.go, handler/skill.go, api.js, views/prompts.js, views/skills.js

## UI 设计方案

### Batch Bar 按钮布局
为避免按钮过多导致布局冲突，采用 **"更多操作"下拉菜单** 方案：

**现有按钮保持不变**：导入、导出、批量删除、退出管理

**新增一个"更多操作"下拉按钮**：点击后展开下拉菜单，包含所有新增的批量操作：

```
┌─────────────────────────────────────────────────────────────┐
│ ☑ 全选  3 项已选  │  导入  导出  [更多操作▾]  批量删除  退出管理 │
└─────────────────────────────────────────────────────────────┘
                          ↓ 点击展开
                    ┌─────────────┐
                    │ 📁 修改分类  │  ← 仅提示词模块
                    │ 🏷️ 添加标签  │
                    │ 🏷️ 移除标签  │
                    │ ⭐ 置顶      │
                    │ ⭐ 取消置顶  │
                    └─────────────┘
```

### 下拉菜单交互规则
- 点击"更多操作"按钮展开下拉菜单
- 点击菜单项执行对应操作（修改分类/添加标签/移除标签需要弹窗）
- 置顶/取消置顶直接执行无需弹窗
- 点击菜单外部区域自动关闭菜单
- 下拉菜单定位在按钮下方，自动边界检测（参考现有 ContextMenu 组件）

## ADDED Requirements

### Requirement: "更多操作"下拉菜单
系统 SHALL 在两个模块的 batch-bar 中提供"更多操作"下拉按钮，点击后展开包含批量操作选项的下拉菜单。

#### Scenario: 展开下拉菜单
- **WHEN** 用户在批量管理模式下点击"更多操作"按钮
- **THEN** 在按钮下方展开下拉菜单，显示可用的批量操作选项

#### Scenario: 关闭下拉菜单
- **WHEN** 用户点击菜单外部区域或点击"退出管理"
- **THEN** 下拉菜单自动关闭

### Requirement: 批量修改分类（仅提示词）
系统 SHALL 在提示词模块"更多操作"菜单中提供"修改分类"选项，允许用户将选中的所有提示词批量设置为指定分类。

#### Scenario: 批量修改分类成功
- **WHEN** 用户选中若干提示词，点击"更多操作"→"修改分类"
- **THEN** 弹出模态框，显示分类输入框（支持新建分类），用户确认后所有选中提示词的分类被更新，列表刷新

#### Scenario: 未选中任何提示词
- **WHEN** 用户未选中任何提示词就点击"修改分类"
- **THEN** 提示"请先选择要操作的提示词"

### Requirement: 批量添加标签
系统 SHALL 在两个模块"更多操作"菜单中提供"添加标签"选项，允许用户为选中的所有项目批量添加标签（不覆盖已有标签）。

#### Scenario: 批量添加标签成功
- **WHEN** 用户选中若干项目，点击"更多操作"→"添加标签"，输入新标签（逗号分隔）
- **THEN** 新标签被追加到选中项目的标签列表中（自动去重），列表刷新

### Requirement: 批量移除标签
系统 SHALL 在两个模块"更多操作"菜单中提供"移除标签"选项，允许用户从选中的所有项目中移除指定标签。

#### Scenario: 批量移除标签成功
- **WHEN** 用户选中若干项目，点击"更多操作"→"移除标签"，选择要移除的标签
- **THEN** 指定标签从选中项目的标签列表中被移除，列表刷新

### Requirement: 批量置顶/取消置顶
系统 SHALL 在两个模块"更多操作"菜单中提供"置顶"和"取消置顶"选项，允许用户批量设置选中项目的置顶状态。

#### Scenario: 批量置顶成功
- **WHEN** 用户选中若干项目，点击"更多操作"→"置顶"
- **THEN** 选中项目的置顶状态设为 true，列表刷新，置顶项排列到顶部

#### Scenario: 批量取消置顶成功
- **WHEN** 用户选中若干项目，点击"更多操作"→"取消置顶"
- **THEN** 选中项目的置顶状态设为 false，列表刷新
